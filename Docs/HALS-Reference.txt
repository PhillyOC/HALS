============================================================
 OVERVIEW
============================================================

HALS (Home Automation & Logging System) is a modular
PowerShell 7 platform that runs on your Windows PC. It
connects to smart-home and network APIs through optional
provider modules, builds a unified device inventory, and
exposes HALSAI — a natural-language assistant that answers
questions about your home and executes device commands
through a policy-checked execution pipeline.

HALS is folder-portable: launch from any copy of the tree
and the session binds to that folder automatically. No
cloud dependency is required for core operation; AI and
device providers are optional add-ons you enable when ready.

What HALS does well today
  - Unified inventory across multiple platforms
  - Plain-English control via Ask-HALSAI
  - Snapshot comparison for network and device changes
  - Local web control panel with REST API
  - OAuth gateway for desktop-friendly provider setup
  - Engineering notebook (HALSLab) for investigations

Current release: 1.0.10
Repository: https://github.com/PhillyOC/HALS


============================================================
 DEVICE PROVIDERS
============================================================

HALS ships eight device provider modules. Each is optional.
Connect only the platforms you use via Initialize-HALSDeviceProvider
or the provider-specific setup command shown at startup.

Provider          Role                          Control
----------------  ----------------------------  -----------------
UniFi             Network clients, APs,         Inventory only
                  gateways, switches

SmartThings       Lights, sensors, locks,       Full device
                  plugs, sirens, and more         actions

Home Assistant    Entities and automations        Inventory
                  visibility                      (extended)

Google Nest       Nest device visibility          Inventory

Philips Hue       Bridge lights, rooms, scenes    Inventory +
                                                    bridge API

WiZ Pro           Official WiZ Pro cloud          Full light
                  building topology               actions

Ecobee            Thermostat visibility           Inventory

Pushbullet        Notification channel            Notify actions

Setup commands (also available from the device wizard)
  Initialize-UniFi
  Initialize-SmartThings
  Initialize-HomeAssistant
  Initialize-GoogleNest
  Initialize-PhilipsHue
  Initialize-WiZ
  Initialize-Ecobee
  Initialize-Pushbullet

Remove an integration:
  Remove-HALSDeviceProvider -Provider SmartThings


============================================================
 AI PROVIDERS (HALSAI)
============================================================

HALSAI is separate from device providers. AI models reason
and produce execution plans; HALS validates permissions and
executes commands through registered provider modules.

Switchable AI providers
  OpenAI         GPT models via API key
  Claude         Anthropic Claude via API key
  Gemini         Google Gemini via API key
  Together AI    Hosted open models via API key
  Mistral        Mistral models via API key
  Ollama         Local models — no API key required

Setup and management
  Initialize-HALSAI              Choose and configure AI
  Initialize-OpenAI              OpenAI wizard
  Initialize-Claude              Claude wizard
  Initialize-Gemini              Gemini wizard
  Initialize-TogetherAI          Together AI wizard
  Initialize-Mistral             Mistral wizard
  Initialize-Ollama              Local Ollama wizard
  Switch-HALSAIProvider          Switch active provider
  Remove-HALSAIProvider          Remove from Config\AI.json

HALS injects the canonical system prompt from
AI\HALSAI-SystemPrompt.txt on every Ask-HALSAI request
and during AI provider initialization.


============================================================
 AUTHENTICATION PATTERNS
============================================================

HALS supports several auth styles. Setup wizards guide you
through each one and store credentials locally under Secrets\.

OAuth (browser authorization + local gateway)
  SmartThings      Default path; desktop httpbin flow or
                   local gateway redirect
  Google Nest      Google OAuth via local gateway
  WiZ Pro          OAuth 2.0 with PKCE
  Ecobee           Ecobee OAuth
  Pushbullet       Pushbullet OAuth

API key / token
  OpenAI, Claude, Gemini, Together AI, Mistral
  Home Assistant   Long-lived access token
  UniFi            ui.com Site Manager API key, or local
                   UniFi OS Integration API key

Local / password
  UniFi            Legacy Cloud Key admin password (ports
                   443/8443 auto-detected)
  SmartThings      Legacy PAT (optional fallback)

Bridge pairing
  Philips Hue      One-time bridge button press to obtain
                   a local API username

OAuth gateway
  HALS auto-starts a local callback gateway on port 8000
  at launch and during OAuth wizards. SmartThings can also
  use Reconnect-SmartThingsOAuth to finish authorization
  when credentials exist but tokens are missing.


============================================================
 STARTING HALS
============================================================

From a HALS folder:

    .\Start-HALS.cmd

or

    pwsh .\Start-HALS.ps1

This loads modules, starts the OAuth gateway, connects to
configured providers, runs a full inventory scan, displays
the device table and provider health, and drops you into
the interactive HALS session.

Rescan at any time:
    HALS


============================================================
 INSTALL OPTIONS
============================================================

  Windows installer     HALS-Setup-<version>.exe from GitHub
                        Releases (no admin required)

  One-line bootstrap    Install-FromGitHub.ps1 from Releases

  Portable zip          Unzip anywhere, run Start-HALS.cmd

  From source           git clone, then Start-HALS.cmd

After install, run Initialize-HALSDeviceProvider and
Initialize-HALSAI to connect platforms and AI.

 COMMAND REFERENCE
============================================================

Commands are available after Start-HALS.ps1 has run.
Type HALSHelp for the in-session quick list.

------------------------------------------------------------
 GET STARTED
------------------------------------------------------------

Initialize-HALSDeviceProvider
  Wizard menu for connecting device platforms.

Initialize-HALSAI
  Wizard menu for choosing and configuring an AI provider.

Reconnect-SmartThingsOAuth
  Finish or refresh SmartThings OAuth when Client ID/Secret
  are saved but tokens are missing or expired.

Switch-HALSAIProvider -Provider <name>
  Switch the active AI provider (OpenAI, Claude, Gemini,
  TogetherAI, Mistral, Ollama).

Remove-HALSDeviceProvider -Provider <name>
  Delete local credential files for a device integration.

Remove-HALSAIProvider -Provider <name>
  Remove an AI provider from Config\AI.json.

------------------------------------------------------------
 EVERYDAY USE
------------------------------------------------------------

Ask-HALSAI "<question or instruction>"
  Natural-language assistant. Answers informational
  questions and executes device commands. Action requests
  run immediately by default. Use -Verbose to preview the
  raw response and execution plan before running.

  Examples:
    Ask-HALSAI "turn the living room lights blue"
    Ask-HALSAI "which lights are on right now?"
    Ask-HALSAI "is the front door locked?"
    Ask-HALSAI "turn the lights purple" -Verbose

HALS
  Full inventory scan across all connected providers.
  Rebuilds the device table, runs the New Device Wizard
  for unknown devices, saves a snapshot, and displays
  provider health.

Knowledge
  Show the Known Device registry (friendly names,
  categories, MAC addresses, tags).

Snapshots
  List saved device snapshots in reverse chronological order.

CompareHALS
  Compare the two most recent snapshots. Reports new
  devices, missing devices, and IP address changes.

Version
  Display the current HALS version.

HALSHelp
  Display the in-session command quick reference.

------------------------------------------------------------
 DEVICE INVENTORY & KNOWLEDGE
------------------------------------------------------------

Get-HALSKnownDevices
  Return Known Device entries as objects for scripting.

Add-HALSKnownDevice
  Manually register a device by MAC address (or HA entity).

Get-HALSInventory
  Return the current unified inventory object.

Get-HALSProviderHealth
  Return health status for all providers (Healthy, Offline,
  NotConfigured, etc.).

Test-HALSProviderHealthy -Provider <name>
  Return True/False for a single named provider.

Save-HALSSnapshot
  Manually save the current inventory to Snapshots\.

------------------------------------------------------------
 COLOR ENGINE
------------------------------------------------------------

Find-HALSColor -Search "<fragment>"
  Search the built-in CSS color database (148 colors).

Get-HALSColor -Name "<color>"
  Return a single color object (Name, Hex, RGB).

 HALSLAB  (Experiments, Observations & Evidence)
------------------------------------------------------------

HALSLab is HALS's built-in engineering notebook for

============================================================
 OAUTH GATEWAY
============================================================

Gateway\HALSGateway.ps1 listens on 127.0.0.1:8000 for
OAuth redirect callbacks. HALS starts it automatically
via Initialize-HALSGateway at launch.

SmartThings-specific commands:
  Initialize-HALSSmartThingsOAuth
  Complete-HALSSmartThingsOAuth -RedirectUrl "<url>"
  Reconnect-SmartThingsOAuth

For SmartThings desktop OAuth, register a public HTTPS
redirect (such as httpbin.org/get) or use tunnel mode
when your OAuth app requires a custom redirect URI.


============================================================
 WEB CONTROL PANEL
============================================================

Start the local web server:

    pwsh .\Web\Start-HALSWeb.ps1

Default URL: http://localhost:8080 (localhost binding only)

The web panel provides a browser UI and REST API for
inventory, scans, snapshots, device actions, and HALSAI.

API routes (prefix /api/)
  GET  /status              Session and provider health
  GET  /overview            Dashboard summary
  GET  /devices             Device list (search/filter)
  GET  /devices/<mac>       Single device detail
  POST /scan                Trigger inventory scan
  GET  /snapshots           List snapshots
  GET  /snapshots/compare   Compare latest two snapshots
  GET  /knowledge           Known device registry
  POST /actions             Execute a provider command
  GET  /ai/providers        List configured AI providers
  POST /ai/ask              Ask HALSAI a question
  POST /ai/execute          Execute an AI plan
  POST /ai/switch           Switch active AI provider

Security note: the web API has no built-in authentication.
Keep the default localhost binding unless you add your own
auth layer before exposing it on a network.


============================================================
 QUICK-START GUIDE
============================================================

Step 1 — Install and launch HALS
  Download from GitHub Releases or clone the repository.
  Run Start-HALS.cmd. HALS scans configured providers and
  shows provider health.

Step 2 — Connect your platforms
  Run Initialize-HALSDeviceProvider and pick a provider.
  Follow the wizard (OAuth browser flow, API key, or local
  pairing as appropriate). Repeat for each platform.

Step 3 — Name your devices
  When the New Device Wizard appears for an unknown device:
    [U]  Keep the current name
    [R]  Rename to something friendlier
    [S]  Skip for now

  Named devices are saved to the Knowledge Base permanently.

Step 4 — Set up HALSAI
  Run Initialize-HALSAI, choose a provider, enter credentials,
  and confirm the connection test. Then try:

    Ask-HALSAI "which lights are on?"
    Ask-HALSAI "turn the bedroom lights warm white"

Step 5 — Explore snapshots and the web panel
  Run CompareHALS after rescans to see what changed.
  Launch Web\Start-HALSWeb.ps1 for a browser dashboard.

Step 6 — Investigate with HALSLab (optional)
  Start-HALSExperiment to track diagnostics alongside your
  home automation work.

============================================================
 LOOKING AHEAD
============================================================

HALS is actively evolving. The modular provider registry
makes it straightforward to add new platforms, expand web
controls, and deepen HALSAI integration as each provider's
command catalog matures. Upcoming work includes richer web
UI coverage aligned with the provider matrix, broader
action support beyond SmartThings and WiZ Pro, and
continued OAuth reliability improvements for desktop setups.

Contributions, provider requests, and feedback are welcome
on GitHub. HALS is MIT-licensed and designed to grow with
your home — one provider at a time.
